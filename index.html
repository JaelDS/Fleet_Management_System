DataStore Design Pattern

The DataStore class serves as a centralized data management pattern, implementing a dictionary-based storage mechanism for O(1) lookup performance.

DataStore Implementation Python 
class DataStore:
    """Base class for data storage
    
    This class provides a foundation for storing, retrieving, and managing data
    using dictionaries. It offers methods to add, update, remove, and find items.
    """
    
    def __init__(self):
        self.data = {}  # Dictionary for O(1) operations
        self.__last_id = 0  # Private attribute (not used for random IDs)
    
    def add(self, item):
        """Add an item with auto-generated ID"""
        # Polymorphic type detection
        if isinstance(item, Vehicle):
            prefix = "V"
        elif isinstance(item, Customer):
            prefix = "C"
        elif isinstance(item, Shipment):
            prefix = "S"
        elif isinstance(item, Delivery):
            prefix = "D"
        else:
            prefix = "I"
        
        # Generate random alphanumeric ID
        alphabet = string.ascii_uppercase + string.digits
        random_part = ''.join(random.choices(alphabet, k=7))
        item_id = f"{prefix}{random_part}"
        
        # Store with unique identifier
        item.id = item_id
        self.data[item_id] = item
        return item_id
Design Rationale:
  • Dictionary vs List: Chose dictionary for O(1) lookup complexity instead of O(n) list traversal
  • Type-Prefixed IDs: Enables instant object type identification (V for Vehicle, C for Customer)
  • Random ID Generation: Prevents predictable sequences and enhances security
  • Centralized Storage: Simplifies data management and maintains single source of truth

1. Encapsulation

Data and methods are bundled within classes, with controlled access through public methods.

Australian Address Validation

Complex regex patterns ensure compliance with Australian address formats, including state codes and postcodes.

Address Validation Pattern Python 
# Comprehensive Australian address validation

Regex validation chosen over simple string checks to ensure data integrity and compliance with Australian postal standards.

Random ID Generation Strategy

Implementation of a secure, collision-resistant ID generation system.

ID Generation Algorithm Python 
# Generate random alphanumeric part
alphabet = string.ascii_uppercase + string.digits
random_part = ''.join(random.choices(alphabet, k=7))

# Create full ID with type prefix
item_id = f"{prefix}{random_part}"  # e.g., "V7A9X2M3"
Security Consideration:

Random IDs prevent sequential guessing attacks and make the system more secure compared to incremental IDs.

Design Patterns

Repository Pattern

The DataStore class implements the Repository pattern, providing a collection-like interface for accessing domain objects.

Pattern Benefits:
  • Centralizes data access logic
  • Provides abstraction over data storage
  • Enables easy switching of storage mechanisms
  • Supports unit testing through dependency injection
